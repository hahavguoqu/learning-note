### Git

#### 1.初始化与创建仓库

`git config --global user.name "hahavguoqu"`    # 设置全局用户名

`git config --global user.email "1048060527@qq.com"`    # 设置全局用户邮箱

`git init `   # 初始化一个新的git仓库

`git clone [url]`    # 克隆到本地



#### 2.本地工作流

`git status`     # 检查仓库状态（哪些文件被修改，哪些在暂存区）

`git add [file]`      # 将文件添加到“暂存区”

`git add .`     # 将添加当前目录所有修改

`git commit -m "Your commit message"`      # 暂存区提交到本地仓库，并附加说明信息

`git log`    # 查看提交历史 

`git log --oneline --graph`    # 提交历史展示更加简洁

`git diff`      # 查看“工作区”与“暂存区”的差异

`git diff --staged `    # 查看“暂存区”和“上次提交”之间的差异

`git restore [file] `   # 撤销工作区中对某个文件的修改（用上次提交的版本覆盖）

`git restore --staged [file]`      # 将文件从暂存区“拿出来”，但保留工作区的修改（即 `git add` 的反向操作）



#### 3.分支管理 

`git branch`     # 列出所有本地分支

`git branch -a`     #列出所有本地和远程分支

`git checkout [branch-name]`     # 切换到指定的分支。

`git checkout -b [new-branch-name]`      # 创建并立即切换到新分支

`git merge [branch-name]`     # 将 `[branch-name]` 分支的更改合并到当前所在的分支



#### 4.远程与协作

`git remote -v`     # 查看配置的远程仓库地址

`git push`      # 本地仓库提交到远程仓库

`git push origin [branch-name]`     # 指定推送到哪个远程仓库的哪个分支

`git pull`     # 从远程仓库拉取最新的更改，并自动与本地分支合并，这相当于 `git fetch` + `git merge`

`git fetch`     # 从远程仓库拉取最新的信息
